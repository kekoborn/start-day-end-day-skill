# Скилл-ассистент планирования дня (`/start-day` + `/end-day`)

Пара [Claude Code](https://claude.com/claude-code) скиллов, которые заменяют ручное планирование дня. Не приложение и не сервис — два markdown-файла с инструкциями, которые Claude Code читает и исполняет как персонального PM.

## Как это работает на практике

Каждое утро прилетает пуш в Telegram: «пора спланировать день». Открываю Claude Code, пишу `/start-day`. Он:
- читает реестр проектов (статусы, приоритеты, дедлайны, роль — исполнитель я или контролёр),
- сверяет его со свежим состоянием календаря,
- предлагает 3–5 тайм-блоков на день с учётом уже стоящих клиентских звонков (их не трогает),
- показывает план и **ждёт моего «ок»** — ничего не создаёт заранее,
- после подтверждения сам расставляет события по нужным календарям (по типу активности — клиентская работа, обучение, буфер и т.д.) и коротко пушит план в Telegram.

Вечером — второй пуш: «пора закрыть день». Пишу `/end-day`. Он:
- сверяет, что из плана реально случилось (по календарю, а не только по моим словам),
- спрашивает: что сделано, что перенесено, что всплыло нового,
- обновляет реестр проектов (статусы, следующие шаги) и дневную заметку,
- **сразу же планирует завтрашний день** — читает завтрашний календарь, предлагает черновик блоков, ждёт «ок»,
- после подтверждения создаёт события на завтра (утренний `/start-day` их уже не пересоздаёт — только проверяет, не сдвинулось ли что-то за ночь),
- пушит мини-ревью дня в Telegram и коммитит изменения реестра/заметок в git.

У меня туда же зашит опциональный модуль: он смотрит финансовую таблицу и, если по задаче стоит статус «выставлен счёт», сверяет, пришла ли оплата — это специфика моей бухгалтерии, в репозитории вынесено отдельным опциональным шагом, который легко выкинуть.

Ключевая идея: один раз настраиваешь доступы (Google Calendar/Sheets OAuth) — дальше просто пользуешься, ничего заново не авторизуешь.

## Что нужно, чтобы развернуть у себя

1. [Claude Code](https://claude.com/claude-code) с поддержкой скиллов.
2. Доступ к Google Calendar API (и, если нужна сверка оплат, Google Sheets API) через OAuth.
3. Markdown-реестр проектов + папка дневных заметок (обычный текстовый файл, никакой базы данных).
4. (опционально) Telegram-бот с бесплатным хостингом (например, render.com) — только для пуш-напоминаний, сам Claude не вызывает.

Ничего из этого не завязано на конкретный вендор — Google Calendar можно дёргать любым способом (CLI, официальный Python-клиент, MCP-коннектор). Ниже — вариант, который проще всего поднять с нуля.

## Пошаговое разворачивание

### Шаг 1 — включить Google Calendar API и получить доступ
1. Зайти в [Google Cloud Console](https://console.cloud.google.com/), создать новый проект (любое имя, например «day-planner»).
2. В разделе **APIs & Services → Library** включить **Google Calendar API** (и **Google Sheets API**, если будешь использовать сверку оплат).
3. В **APIs & Services → Credentials** создать **OAuth client ID** типа *Desktop app*. Скачать JSON с ключом — он понадобится тому инструменту, которым будешь дёргать API (см. шаг 2).
4. Если приложение останется в статусе *Testing* — раз в 7 дней Google будет просить переавторизоваться. Чтобы этого избежать, в **OAuth consent screen** можно опубликовать приложение (для личного использования это ок, экран верификации показывать не будет — предупреждение можно игнорировать).

### Шаг 2 — выбрать способ говорить с Calendar/Sheets из Claude
Нужен любой инструмент, который после OAuth-авторизации умеет читать/создавать события и (опционально) читать таблицы, и которым Claude Code может пользоваться через Bash. Варианты:
- **CLI-обёртка** (то, что использую я) — консольная утилита, которая один раз проходит OAuth, кладёт токен в системный keyring, дальше все команды идут без повторной авторизации. Если пишешь такую сам — минимум нужны `calendar.events.list`, `calendar.events.insert`, `calendar.events.patch` (и `spreadsheets.values.get`, если нужна сверка оплат).
- **Официальный Google API клиент** (`google-api-python-client` + `google-auth-oauthlib`) — небольшой Python-скрипт на 30–40 строк, тоже кэширует токен в файл после первого запуска.
- **MCP-коннектор Google Calendar**, если он у тебя стабильно работает (у меня были обрывы соединения, поэтому я ушёл на CLI).

Какой бы способ ни выбрал — команды из `start-day/SKILL.md` и `end-day/SKILL.md` (раздел «Календарь») написаны как псевдо-CLI (`gws calendar events list/insert/patch`) — замени их на реальные команды/скрипт своего инструмента, логика (что читать, когда создавать, когда патчить) остаётся той же.

### Шаг 3 — получить ID своих календарей
Для каждого календаря, который хочешь использовать (клиентские звонки, рабочие блоки, обучение, еда и т.д.):
1. Google Calendar → шестерёнка у календаря в левом меню → **Settings and sharing**.
2. Раздел **Integrate calendar** → скопировать **Calendar ID** (для основного календаря это обычно твой email, для остальных — что-то вида `xxxxx@group.calendar.google.com`).

Подставь эти ID вместо `<CLIENT_CALENDAR_ID>`, `<WORK_CALENDAR_ID>` и т.д. в таблице «Календари» в обоих SKILL.md.

### Шаг 4 — завести реестр проектов и дневные заметки
Создай папку, например `~/day-planner/`, а в ней:
```
day-planner/
├── projects/_registry.md
└── Daily Notes/
```
Формат строки в реестре:
```
### 🟢 Имя проекта · `slug`
- Статус: active|waiting-os|hold|done · Здоровье: 🟢|🟡|🔴 · Приоритет: P1|P2|P3|hold
- Следующий шаг: ...
```
Дневная заметка `Daily Notes/2026-07-10.md` создаётся автоматически скиллами (разделы `## 🗓 План` и `## 🌆 Итоги`) — вручную ничего заводить не нужно, кроме самой папки.

### Шаг 5 — поставить сами скиллы
```bash
mkdir -p ~/.claude/skills/start-day ~/.claude/skills/end-day
cp start-day/SKILL.md ~/.claude/skills/start-day/SKILL.md
cp end-day/SKILL.md ~/.claude/skills/end-day/SKILL.md
```
Открой оба файла и замени плейсхолдеры:
- `<PROJECT_ROOT>` → путь к своей папке (например `~/day-planner`)
- `<CLIENT_CALENDAR_ID>`, `<WORK_CALENDAR_ID>`, `<COMMS_CALENDAR_ID>`, `<LEARNING_CALENDAR_ID>`, `<MEALS_CALENDAR_ID>` → ID из шага 3
- `<Your/Timezone>` → свой часовой пояс, например `Europe/Moscow`
- команды `gws ...` → команды своего инструмента из шага 2

Если сверка оплат не нужна — удали раздел «Сверка поступлений/оплат» в `start-day/SKILL.md` целиком (шаг 0 в разделе «Шаги» тоже).

### Шаг 6 — первый прогон
В Claude Code (в любом чате) написать `/start-day`. Первый раз плана в заметке ещё нет, поэтому он пойдёт в полный режим: соберёт реестр → прочитает календарь → предложит блоки → дождётся «ок» → создаст события. Проверь, что события легли в нужные календари с нужными цветами.

Вечером — `/end-day`, чтобы проверить вторую половину цикла (сверка факта + планирование завтра).

### Шаг 7 (опционально) — Telegram-пуши
Это просто напоминания по расписанию, сам Claude их не отправляет и не слушает:
1. Создать бота через [@BotFather](https://t.me/BotFather), получить токен.
2. Задеплоить небольшой скрипт (Python/Node), который по cron дважды в день шлёт тебе в Telegram текст-напоминание («пора спланировать день» / «пора закрыть день») через Bot API (`sendMessage`). Хостить можно бесплатно, например на render.com (cron job или long-running процесс с внутренним таймером).
3. Пуш — это только триггер тебе открыть Claude Code и написать `/start-day`/`/end-day`; сам скилл о существовании бота не знает. Если хочешь также **получать** итог дня в Telegram (а не только напоминание), последний шаг обоих SKILL.md — отправка сообщения через тот же Bot API (`sendMessage` с текстом плана/итогов) тем же скриптом или отдельным вызовом `curl` из скилла.

### Шаг 8 — донастройка под себя
- Правила приоритизации (раздел «Приоритизация» в `start-day/SKILL.md`) — подстрой порядок под свою работу.
- Таблица «Календари» — добавь/убери категории под свои типы задач.
- Если ведёшь канбан отдельно от реестра — впиши свой путь и формат карточек, если нет — просто убери упоминания канбана из шагов.
- Коммит в git (последний шаг `end-day`) — опционален, нужен только если `<PROJECT_ROOT>` — git-репозиторий.

Настраивается один раз (шаги 1–5) — дальше просто `/start-day` утром и `/end-day` вечером.
