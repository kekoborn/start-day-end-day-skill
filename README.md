# Скилл-ассистент планирования дня (start-day / end-day)

Пара Claude Code скиллов, которые заменяют ручное планирование дня:
- **`/start-day`** (утро) — сверяет вчерашний план с календарём, приоритизирует задачи из реестра проектов, предлагает тайм-блоки, после твоего «ок» создаёт события в Google Calendar и пушит план в Telegram.
- **`/end-day`** (вечер) — сверяет факт с планом, обновляет реестр проектов, пишет итоги дня в заметку, **уже планирует завтрашний день** (утренний `/start-day` его только проверяет и стартует), пушит мини-ревью в Telegram, коммитит изменения в git.

Это не готовый продукт — это персональный workflow, обобщённый под шаринг. Файлы `start-day/SKILL.md` и `end-day/SKILL.md` — рабочий скелет, в них расставлены `<ПЛЕЙСХОЛДЕРЫ>` вместо моих личных ID/путей. Чтобы это заработало у тебя, нужно самому:

## Что нужно подключить

1. **Claude Code** с поддержкой скиллов (`~/.claude/skills/<name>/SKILL.md`).
2. **Доступ к Google Calendar/Sheets.** Я использую CLI [`gws`](https://github.com) (один бинарь на Drive/Docs/Sheets/Slides/Calendar/Gmail, креды хранятся в keyring, не нужно каждую неделю переавторизовываться). Альтернатива — Google Calendar/Sheets MCP-коннектор, но у меня он был нестабилен (`net::ERR_FAILED`), поэтому ушёл на CLI. Тебе нужно один раз пройти OAuth и получить ID своих календарей / ID гугл-таблиц.
3. **Markdown-реестр проектов** — файл вида `projects/_registry.md` со статусами/приоритетами/дедлайнами по твоим проектам, плюс дневные заметки `Daily Notes/<YYYY-MM-DD>.md`. Формат — ниже.
4. **(опционально) Telegram-бот** — чтобы план/итоги дня прилетали пушем. У меня захостен бесплатно на render.com, дёргается через Bot API. Без этого пункта скиллы всё равно работают — просто не будет пуша, план останется в заметке.
5. **(опционально) Логика сверки оплат** — у меня зашита сверка «касса ↔ выставленные счета» под мою бухгалтерию (agency-специфика). Это отдельный модуль внутри `start-day` — либо адаптируй под свою таблицу, либо вырежи целиком, если не нужно.

## Формат реестра проектов (пример строки)

```
### 🟢 Имя проекта · `slug`
- Статус: active|waiting-os|hold|done · Здоровье: 🟢|🟡|🔴 · Приоритет: P1|P2|P3|hold
- Следующий шаг: ...
```

## Формат дневной заметки

```
## 🗓 План
10:00–12:00 — #slug: действие [исполнитель/контролёр]
...

## 🌆 Итоги
- ✅ Сделано: ...
- ⏭ Перенесено: ...
- 🆕 Всплыло: ...
- 📌 На завтра: ...
```

## Как разворачивать у себя

1. Скопируй `start-day/SKILL.md` → `~/.claude/skills/start-day/SKILL.md`, `end-day/SKILL.md` → `~/.claude/skills/end-day/SKILL.md`.
2. Замени все `<ПЛЕЙСХОЛДЕРЫ>` на свои пути/ID (реестр, дневные заметки, календари, опционально таблица оплат, опционально Telegram-бот).
3. Первый запуск — дай доступ к Google Cloud Console (OAuth) один раз, дальше `gws` сам переиспользует токен.
4. Дальше просто вызывай `/start-day` утром и `/end-day` вечером (или подключи те же Telegram-пинги, что и я).

Настраивается один раз — дальше просто пользуешься.
